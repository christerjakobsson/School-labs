Exercise: Scheduling

Prerequisites

Before you start, you should do the following:

  1. Attend the lecture on scheduling.

Objectives

After completing this exercise you should be able to

  1. Define a theoretical task graph scheduling model.
  2. Schedule small task graphs on paper using a list scheduling algorithm.
  3. Define and motivate the critical path heuristic

Instructions

Scheduling tasks

Uniform costs

Suppose you have a collection of tasks with uniform cost.

How would you schedule these tasks to minimize the execution time?

Non-uniform costs

Now consider the following collection of tasks and varying task costs: 

A: 3
B: 2
C: 5
D: 3
E: 4
F: 1
G: 3
H: 2

Answer the following questions:

  • How long will it take a single processor to complete these tasks, i.e., what is the total work?
  • What do you get if you schedule the tasks on three processors starting with the shortest task?
  • What do you get if you schedule the tasks on three processors starting with the longest task?
  • How would you schedule these tasks on three processors to minimize the execution time?

Scheduling task graphs

Suppose that the tasks are not completely independent but are constrained by a task graph.

Come up with a scheduling algorithm that you think will do better than random scheduling.

Come up with a scheduling algorithm that you think will do worse than random scheduling.

(How would you test if your algorithms actually perform better/worse than random scheduling?)

Study the classic article "Bounds for Certain Multiprocessing Anomalies" (see the Resources section below).
