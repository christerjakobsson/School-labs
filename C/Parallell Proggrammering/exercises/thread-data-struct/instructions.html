Exercise: Concurrent data structures

Prerequisites

Before you start, you should do the following:

  1. Attend the lecture on synchronization.
  2. Complete the exercise on barriers and latches.

Objectives

After completing this exercise you should be able to

  1. Write and use lock-based concurrent data structures.

Instructions

A concurrent data structure can be safely used by several threads in parallel. Examples include concurrent stacks, queues, sets, trees, graphs, hash tables, etc.

Concurrent data structures can and often are implemented using locks and condition variables.

In this exercise, you will write your own concurrent data structure.

The data structure you are about to implement can be called an exchange. It is used by exactly two threads to synchronously send data from one thread to the other.

The exchange data structure supports two operations:

put
waits for the receiver and then sends the data.
get
waits for the sender and then receives the data.

You can, for example, use the exchange to implement the producer-consumer pattern:

  • The producer uses put to send produced data to the consumer.
  • The consumer uses get to receive produced data from the producer.

Design the data structure and its interface on paper. Focus on the use of locks and condition variables to synchronize the threads.

Compare your design with your class mates' designs.

Optionally implement and test the design.
