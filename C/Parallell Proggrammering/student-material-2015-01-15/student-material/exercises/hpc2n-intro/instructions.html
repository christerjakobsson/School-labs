Exercise: Sign in to HPC2N for the first time

Prerequisites

Before you start, you should do the following:

  1. Obtain an HPC2N account and password (handed out during the first lecture; ask a teacher if you did not attend).
  2. Install an SSH client (e.g., ssh in Linux and putty in Windows) for secure remote shell access.

Objectives

After completing this exercise you should be able to

  1. Sign in to the Abisko cluster at HPC2N using SSH.
  2. Change your password.
  3. Forward your HPC2N email.

Instructions

The following instructions cover only Linux systems using ssh for remote access. If you use some other system or client, then you need to consult its documentation yourself.

Sign in to Abisko

Sign in to Abisko using the command 

ssh yourusername@abisko.hpc2n.umu.se

and then enter your default password. If you entered the correct username and password, then you should now be signed in. If you enter the wrong password three times in a row, then your account will be temporarily frozen and you will have to contact one of the teachers to have it unlocked again, so please be careful when you write your password!

Change your password

The first thing you must do is to change your password using the command 

passwd

Failure to do so during your first login may result in your account being locked.

Forward your HPC2N email

Your HPC2N account has its own email address to which email emitted by HPC2N (mostly the batch system) will be sent. You should therefore forward this email to an address that you check regularly. To do so, write your alternate email address on a single line in a file called ~/.forward, for example using the command 

echo "youremail@cs.umu.se" > ~/.forward
