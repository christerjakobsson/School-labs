Exercise: Create and join threads

Prerequisites

Before you start, you should do the following:

  1. Refresh your skills in C programming and Linux commands
  2. Attend the lecture on thread programming

Objectives

After completing this exercise you should be able to

  1. Create a new thread.
  2. Synchronize with the completion of a thread (i.e., join).

Instructions

  • You will need to use two thread API functions (see Resources) so begin by reading their man pages.
  • Include the system header pthread.h
  • Create a small program that prints "Hello" from one thread and "World" from another thread.
  • Compile and link with the flag -lpthread.
  • What do you think will happen if you run the program multiple times? Will you always get the same output? Why/why not?
  • Try it out! What happened in practice?
  • Now measure the time it takes to create and join one thread using a Timing function.
  • Run the program several times.
  • How long does it take on average to create and join a thread?
  • Do you consider this to be cheap or expensive? To get a feeling you can estimate how many instructions a core can execute in the same amount of time.

Resources

Useful functions:

  • pthread_create
  • pthread_join
  • gettimeofday

Timing function

Using gettimeofday, you can construct a useful timing function that returns the time in seconds with high precision: 

#include <sys/time.h>

double gettime(void)
{
    struct timeval tv;
    struct gettimeofday(&tv, NULL);
    return tv.tv_sec + 1e-6 * tv.tv_usec;
}

This function will be useful in many of our exercises and also some labs.
