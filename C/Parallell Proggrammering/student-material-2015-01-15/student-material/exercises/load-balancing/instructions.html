Exercise: Load balancing

Prerequisites

Before you start, you should do the following:

  1. Attend the lecture on load balancing and termination detection.

Objectives

After completing this exercise you should be able to

  1. Theoretically analyze the load (im)balance of a parallel program.
  2. Choose or design an appropriate load balancing scheme for a particular application.
  3. Give examples of static, dynamic, and adaptive load balancing schemes.

Instructions

Analysis and design of load balance schemes

Example

Consider this program: 

for (int k = 0; k < n; ++k)
{
    f();
}

The iterations are independent and each call to f takes a constant amount of time.

How many different ways can you come up with to distribute the iterations to threads in such a way that the load is balanced or nearly so?

Example

Consider this program: 

for (int k = 0; k < n; ++k)
{
    f(k);
}

The iterations are independent and each call to f takes time proportional to the magnitude of the argument.

How many different ways can you come up with to distribute the iterations to processors in such a way that the load is balanced or nearly so?

Example

Consider this program: 

for (int k = 0; k < n; ++k)
{
    f();
}

The iterations are independent and each call to f takes a random amount of time.

How many different ways can you come up with to distribute the iterations to processors in such a way that the load is balanced or nearly so?

Example

Consider this program: 

for (int k = 0; k < n; ++k)
{
    f(k);
}

The iterations are independent and each call to f takes a time proportional to the magnitude of the argument plus some relatively small positive random number.

How many different ways can you come up with to distribute the iterations to processors in such a way that the load is balanced or nearly so?
